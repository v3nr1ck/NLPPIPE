Here is the precise architectural approach and tooling required to solve this. 
The Architecture: Constrained Generation 
You cannot rely on a standard LLM prompt to reliably output exact, locked unique IDs 100% of 
the time. You must use Constrained Generation (also called Structured Generation). 
Constrained generation intercepts the LLM's output at the token level and mathematically forces 
it to only generate text that matches a specific JSON schema or Regex pattern that you define. 
Workflow: 
1.​ Ingest: Adapter receives client payload (e.g., {"asset": "Chiller 2", "issue": "freon leak, 
pressure dropping"}). 
2.​ Contextualize (Optional but recommended): If your internal database of "Problem 
Codes" is massive (thousands of rows), run a fast vector search (using 
Sentence-Transformers) on the word "freon leak" against your internal database to 
retrieve the top 10 most relevant internal Problem Codes. 
3.​ Prompt Construction: Pass the client payload and the allowed internal IDs to the LLM. 
4.​ Constrained Inference: The LLM evaluates the text, but the inference engine physically 
restricts its output to your exact internal unique IDs. 
5.​ Output: A guaranteed-valid JSON payload mapping to your 4 locked fields.

from unsloth import FastLanguageModel 
import torch 
from trl import SFTTrainer 
from transformers import TrainingArguments 
from datasets import load_dataset 
 
# 1. Load Model (4-bit for speed) 
model, tokenizer = FastLanguageModel.from_pretrained( 
    model_name = "unsloth/mistral-7b-v0.3-bnb-4bit", 
    max_seq_length = 2048, 
    load_in_4bit = True, 
) 
 
# 2. Add LoRA Adapters (The "Fine-Tuning" layer) 
model = FastLanguageModel.get_peft_model( 
    model, 
    r = 16,  
    target_modules = ["q_proj", "k_proj", "v_proj", "o_proj"], 
    lora_alpha = 16, 
    lora_dropout = 0, 
) 
 
# 3. Load Dataset 
dataset = load_dataset("json", data_files="dataset.jsonl", split="train") 
 
# 4. Set Training Arguments 
trainer = SFTTrainer( 
    model = model, 
    tokenizer = tokenizer, 
    train_dataset = dataset, 
    dataset_text_field = "text", # Ensure your JSONL maps to a 'text' prompt template

    max_seq_length = 2048, 
    args = TrainingArguments( 
        per_device_train_batch_size = 4, 
        gradient_accumulation_steps = 4,

Calculating Confidence ($C$) 
You can represent confidence simply as the mean of the softmax probabilities ($P$) for the 
generated tokens ($N$): 
$$C = \frac{1}{N} \sum_{i=1}^{N} P_i$$ 
In your Python code (using the OpenAI-compatible API from Ollama or vLLM), you can request 
logprobs=True. This gives the users a "percentage" they can understand.
